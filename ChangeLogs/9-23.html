Home Page (CSS)

  • In general, I wanted the entire website to have a consistent theme that looked modern and sleek. I wanted the homepage to only have an intro of my portfolio and links to the game, change-log/TODO, and Source Code pages.
  • The first step I took to have a uniform website theme was creating a CSS stylesheet (index.css) that I will link to every page. This will have my fonts, colors, and classes of how I want each element to look like on my website.
  • I wanted a flat font that was easy to read so I learned how to import Google fonts into a CSS file and used the Lato font for the entire website.
  • I then learned how to create CSS classes to specify how I want each element to look like in my website. For example, specifying how I want all of my headers to look.
  • I wanted the homepage to only have links to the content for my website so I created both links on the homepage itself and a side-menu that includes all of the other pages as well.
  • To create the side-menu that I wanted in my head, I had to learn some CSS animations. For the hamburger icon, I learned that you can use "transform: " in CSS to rotate elements. I simply created two lines and rotated them in an X shape when it was active. I then learned about "transition: " which creates this smooth transformation given how much time you want the transformation to take. I combined these two elements to create the hamburger icon animation which turns into an X once active.
  • I figured out that I could have different parts of my website outside of the view of the page. So I essentially have the side-menu at -400px off the screen to the right and when the menu-icon is clicked, the side-menu then slides over 400px to show up on the screen. I also had to set all of the other elements to move to the left 400px to have everything in uniform. Pairing this with the "transition" property, I was able to create a smooth sliding animation for when the menu appears.
  • All of the links still looked static, so I learned how to change the link colors when the mouse hovers over them by defining a class using "::hover". This allows me to change the color, add animation, etc. everytime the mouse hovers over a link.
  • The biggest challenge for the redesign wasn't the programming but it was finding aesthetic and readable colors to make the website look good. For example, I knew I wanted a dark theme for my website but I couldn't use straight white because it strained the eyes when reading text. So I had to use an off-white color. Just finding the colors that matched with each other took some experimenting and research.

Change Log/TODO Page (CSS)

  • At first, I was just going to have the Change Log page be static text with the colors of the website theme... Until I found out about CSS animation libraries.
  • I may have overused some animations but I thought I'd spice up the boring Change Log/TODO page.
  • I was doing some googling and found this website https://animate.style which had a lot of CSS animations already coded for.
  • I wanted to include the animations from this website to the headers of my change log page to create this "loading" animation just so that it's not static and for me to learn how to do this.
  • At first I included the animations into my css stylesheet as their documenation said I should but nothing was working. I fell into a rabbit hole of trying to debug this for almost two hours. I gave up for a second, but as a last ditch effort, I changed where I linked the library from my CSS file to the HTML file and it worked somehow! I don't know why, but I had to link the library in each HTML file that I wanted to use these animations in.
  • I then learned a cross-out effect on my TODO list because I wanted it to behave like an interactive notebook almost. To achieve this animation I had to create a rectangle around the text but made the size to be 1px and set the magins so that the bottom of the rectangle was placed in the middle of the text to create a line. "There might be an easier way to do this like a library, but I just decided to try to create it on my own."
  • I also had to use Keyframes in this strike-through animation to adjust the speed so it wasn't going too fast to where you can't even see the animation. I am familiar with Keyframes in Adobe and other video-editing software so this didn't take me too long to understand.
  • I wanted the finished items in the TODO list to be crossed out and reveal text when I hovered over it. So I included an animation property on the strike-through line using the cubic-bezier property to have this smooth looking animation of the line revealing the entire line of text. I used this website https://cubic-bezier.com to experiment with the parameters to see which speed and movement I liked for the animation.
